我的博客构建流程
发布于：2019-05-16 22:38:00 标签：/ Github / Hexo / 博客构建 / 访问：

前言

  “Anu的博客”到目前为止，构建已经差不多结束了（撒花）！搜索、评论、统计，基本功能几乎都有了，接下来就是更新博客了！

  这一篇打算讲讲我构建这样一个博客的大概流程，讲讲帮助我构建这个博客的各个工具和文章，不会太详细，大概就列举一下博客含有的功能与其构架方法。

博客构建流程

一、Github

  我用了一下午了解了Github的相关知识，其中对我比较有帮助的有:

  • 菜鸟教程-Git教程
  • Git-百度百科
  • Github-百度百科

  看完这些，对Github及其使用就有一个基本的了解了。当然，最重要还是去Github上自己建一个测试用仓库尝试一下各种操作（建议找一个小伙伴一起测试分支与合并功能！）。

  了解完Github后，我在Github上闲逛时意外发现了朋友的github.io博客，当机立断决定也做一个github.io博客来玩。

二、Hexo

  构建过程我参考了：

  • 我是如何利用Github Pages搭建起我的博客，细数一路的坑

  这位博主写的很详细，让我少走不少弯路。我与他选择了不同的主题，之后的配置也不同。

三、Theme-AirCloud的配置

  我使用的Hexo是AirCloud,作者大大在Github上写了很详细的初始化配置过程（万分感谢！）。配置过程并不是很困难，对与新手来说也很友好。在熟悉后我甚至尝试去改变了主题布局，对一个准程序员来说摸索一下并不是很困难。

  在这个主题的基础上，我添加了一些功能，便于博客的使用。

3.1 Hexo Admin

  为了更便捷的发布博客，我找到了Hexo Admin这个方法。Hexo Admin是一个Hexo博客的可视化管理界面，比起直接修改文件，这个方法要便捷许多（但有时不得不修改源文件…）。



  Hexo Admin的安装参考：

  • 如何优雅地发布Hexo博客

  这个教程同样很详细，但其中有一个关于deployCommand文件的坑。我尝试了博主的.sh文件方式发现会出现未知异常，搜索一下发现这个是Linux系统的脚本文件，在Windows下使用有些滑稽了。解决方法就是在根目录下新建一个这样的.bat文件：

  之后将在_config.yml文件中添加的deployCommand标签属性改为(‘hexo-deploy.bat’)，即可成功使用Hexo Admin的Deploy操作。

  需要注意的是，这里虽说是Deploy方法，但我并没有在bat文件中添加”hexo d”命令，一般在修改完博客最后，再手动在Git bash中执行hexo deploy指令。当然如有需要也可以在bat中添加hexo d指令。

3.4 其他

  评论功能我使用了主题内置的Livere，可在墙内使用，在主题配置的Wiki中有写到这一部分。这个Livere有和百度统计相似的评论统计和管理界面，使用也比较方便。当然也可以自己摸索，选择自定义的评论组件。
