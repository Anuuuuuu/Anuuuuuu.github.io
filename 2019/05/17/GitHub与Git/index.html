1.1.Git是什么？Github是什么？

  在我目前的理解里（可能有误），Github就是一个使用了Git作为版本控制系统的云平台。

  首先看一下Git的工作流程。

  Git的工作流程

  Git的操作流程如下：

  • 克隆 Git 资源作为工作目录。
  • 在克隆的资源上添加或修改文件。
  • 如果其他人修改了，你可以更新资源。
  • 在提交前查看修改。
  • 提交修改。
  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

  Git工作基本原理就是在本地选择一个文件夹作为本地仓库，将工程资源从云上拉下来到本地仓库，并在本地对工程进行修改，修改完毕后上传。

  使Git区别于其他版本控制系统的是它的分支（branch）管理，这个分支管理允许工程参与者创建不同的分支，可以理解为往不同的功能方向进行更新，就如同时间线在一个节点发生变化，或者理解为点了不同技能树分支（这样解释会不会很中二？）。Git提供了分支合并（merge）操作，可以将不同的分支整合到一起。当分支间发生冲突时，会自动整合一部分，并可由主开发者手动选择整合，很方便。

  而Github则提供了这样一个存放工程资源的云平台。

  GitHub是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库格式进行托管，故名GitHub。
